# buckaroo-js-core — built-package integration test

A tiny external consumer that installs the **packed** `buckaroo-js-core` tarball into its own `node_modules`, then renders a `DFViewer` and asserts the cells are in the DOM via Playwright.

The purpose is to catch any way the published npm package could be broken without the in-workspace tests noticing — missing `main`/`module`, mis-typed `exports`, stripped `dist/` files, peer-dep regressions, CSS subpath that fails outside the workspace, etc.

## Why this lives outside `packages/`

The Buckaroo pnpm workspace is defined at `packages/pnpm-workspace.yaml`. If this directory lived under `packages/`, pnpm would symlink `buckaroo-js-core` from the workspace and we'd silently be testing source again. By living at `tests/integration-tests/built-pkg/` (outside `packages/`) and using **plain `npm install`** (not pnpm), there is no workspace path to take.

The Playwright spec also asserts that no resource fetched during the page load resolves under `packages/buckaroo-js-core/src/` as a belt-and-suspenders check.

## Run it

From the repo root:

```sh
bash tests/integration-tests/built-pkg/run.sh
```

What that does:

1. `pnpm install` + `pnpm run build` for `packages/buckaroo-js-core` (produces fresh `dist/`).
2. `pnpm pack` produces `buckaroo-js-core-<version>.tgz`, renamed to a fixed `buckaroo-js-core.tgz` so this directory's `package.json` can reference it without tracking the version.
3. `rm -rf node_modules package-lock.json` then `npm install --no-fund --no-audit` inside this directory. The `buckaroo-js-core` dep is `file:./buckaroo-js-core.tgz`, so npm extracts the tarball into `node_modules/buckaroo-js-core/`.
4. `playwright install chromium` (idempotent).
5. `npm test` runs the Playwright spec. The Playwright config's `webServer` boots Vite (`npm run dev`) on `127.0.0.1:5174`.

## What the spec asserts

`pw-tests/renders.spec.ts`:

- **`page title and three sample rows appear in the DOM`** — `DFViewer` renders cells; the values `alpha` / `beta` / `gamma` from `src/main.tsx` and the numeric `22` are all visible.
- **`imports resolve to the installed tarball, not workspace source`** — no observed URL contains `packages/buckaroo-js-core/src`. (Visibility into `node_modules` URLs is best-effort because Vite pre-bundles deps; the hard failure is on workspace-source leaks.)

## Files

```
tests/integration-tests/built-pkg/
├── package.json           # plain-npm consumer; buckaroo-js-core = file:./...tgz
├── vite.config.ts         # pinned to 127.0.0.1:5174 so Playwright reaches it
├── tsconfig.json
├── index.html
├── src/main.tsx           # imports { DFViewer } from "buckaroo-js-core"; renders
├── playwright.config.ts   # webServer = npm run dev
├── pw-tests/renders.spec.ts
├── run.sh                 # one-shot orchestrator
├── .gitignore             # excludes node_modules, the tarball, dist/, reports
└── README.md              # this file
```

## What is NOT tracked

`node_modules/`, `package-lock.json`, the packed `buckaroo-js-core*.tgz`, `dist/`, and Playwright `test-results/` / `playwright-report/`. Everything is regenerated by `run.sh`.

## When to run this

- Manually before publishing `buckaroo-js-core` to npm.
- Any time `packages/buckaroo-js-core/package.json` exports, files, or main/module fields change.
- After dependency bumps to `react` / `ag-grid` / `vite` that could affect the bundle shape.

Not currently part of CI — add to the release workflow if you want a pre-publish gate.
